EARTHQUAKE PREDICTION USING LSTM MODELS
Earthquake Sequence Mining with Dual LSTM Architecture

Author: Haind

Date: 26/03/2025

Version: 2.0

1. PROJECT OVERVIEW

This project focuses on building an earthquake prediction system using two specialized LSTM models. The first model, TimeLSTM, performs binary classification to predict the likelihood of an earthquake occurring within the next seven days. The second model, MagLSTM, performs regression to predict the exact magnitude of the next earthquake. Both models are trained on the same global earthquake dataset from 2000 to 2025 with approximately 100,000 events.

2. DATA PROCESSING PIPELINE

Global earthquake data from the USGS catalog (2000-2025) is processed through a multi-layer pipeline to generate features suitable for LSTM models. Data is divided by geographic regions (0.5° grid), sorted by time, and enriched with physics-based features and temporal features. Finally, two separate datasets are created: 21 features for TimeLSTM (binary classification) and 26 features for MagLSTM (regression).

3.2 Feature Categories

Original Features (4-9) - Raw data from USGS catalog: mag, depth, sig, region_code (both models), latitude, longitude, mmi, cdi, felt (MagLSTM only). Region_code is calculated from 0.5° grid (~55km) to group spatially proximate events.

Core Features (5) - Physics-based computed features: is_aftershock (based on Omori's law), mainshock_mag, seismicity_density_100km (density within 100km radius), coulomb_stress_proxy (estimated accumulated stress from 20 most recent events), regional_b_value (Gutenberg-Richter index).

Sequence Features (6) - Features from spatio-temporal clustering (DBSCAN: <50km + <72h): sequence_id, seq_position, is_seq_mainshock, seq_mainshock_mag, seq_length, time_since_seq_start_sec. These features allow the model to learn earthquake sequence patterns.

LSTM Temporal Features (7) - Special temporal features for LSTM: time_since_last_event, time_since_last_M5, interval_lag1-5. These are the primary LSTM inputs, enabling the model to learn complex temporal patterns from the history of the 5 most recent events.

4. MODEL ARCHITECTURE

Figure 1: Dual LSTM Architecture for Earthquake Prediction

Figure 1 illustrates the Dual LSTM architecture with two parallel processing branches. TimeLSTM (upper branch) performs binary classification to predict the probability of an earthquake occurring within 7 days. MagLSTM (lower branch) performs regression to predict the exact magnitude of the next earthquake. Both branches share the same input data but use different feature sets optimized for each task.

Both models use similar architecture: two LSTM layers (128 → 64 units), Dropout 0.3, Dense 64-64 with ReLU. TimeLSTM outputs probability through sigmoid, using BCEWithLogitsLoss with pos_weight to handle class imbalance. MagLSTM outputs magnitude directly, using MSELoss. Both use Adam optimizer with learning rate 0.001 and ReduceLROnPlateau scheduler.

5. TRAINING DATA STRUCTURE

The two models use separate datasets optimized for each task's characteristics. TimeLSTM uses 21 features focusing on temporal patterns and sequence information. MagLSTM uses 26 features including both temporal and spatial features (latitude, longitude, regional b-value). Both models employ temporal split 80-10-10 (80% training, 10% validation, 10% test) to ensure real-world scenario evaluation. Sequence length is 5 timesteps, allowing models to look back at the 5 most recent events.

7. SYSTEM ARCHITECTURE

7.1 Data Processing Pipeline

The data processing pipeline is designed as a 7-layer sequential pipeline, transforming raw data into training-ready features. Input data is the earthquake catalog from USGS with approximately 100,000 events from 2000-2025. After processing, data is split into two separate datasets for the two models with different feature counts.

Layer 1: Data Loading & Preprocessing - Raw earthquake data is loaded and filtered by time period. Region_code is calculated based on a 0.5° geographic grid (~55km at equator), then data is sorted by geographic region and time. This sorting ensures temporal order within each region, essential for time-series problems.

Layer 2: Spatial Indexing - Spatial index is built using KD-Tree structure to accelerate spatial queries. Distances to neighboring events are calculated and stored, enabling rapid identification of spatially proximate events.

Layer 3: Spatio-temporal Clustering - Events are clustered using DBSCAN algorithm with two thresholds: 50km for spatial distance and 72 hours for temporal distance. Events close in both space and time are grouped into the same sequence, allowing the model to learn earthquake sequence patterns.

Layer 4: Physics-based Feature Extraction - Physics-based features are calculated: aftershock detection using Omori's law (aftershock frequency decreases over time), seismicity density within 100km radius, Coulomb stress proxy estimating accumulated stress from the 20 most recent events, and regional b-value according to Gutenberg-Richter formula.

Layer 5: Temporal Feature Engineering - Temporal features for LSTM are created: time_since_last_event (time since the most recent event), time_since_last_M5 (time since the most recent M5+ event), and interval_lag1-5 (intervals to the 5 previous events). These features allow LSTM to learn complex temporal patterns.

Layer 6: Target Variable Creation - Two target variables are created for different tasks: target_quake_in_7days (binary classification - earthquake within 7 days or not) for TimeLSTM, and target_next_mag (regression - next earthquake magnitude) for MagLSTM.

Layer 7: Data Split & Export - Data is split chronologically (temporal split) with 80-10-10 ratio: first 80% for training, next 10% for validation, final 10% for test. Two separate datasets are exported: features_time.csv (21 features for TimeLSTM) and features_mag.csv (26 features for MagLSTM).

7.2 Model Architecture

TimeLSTM Architecture - Binary classification model with architecture: Input Layer (21 features × 5 timesteps) → LSTM Layer 1 (128 units, dropout 0.3) → LSTM Layer 2 (64 units, dropout 0.3) → Dense Layer 1 (64 units, ReLU) → Dense Layer 2 (64 units, ReLU) → Output Layer (1 unit, logits). Sigmoid activation is applied to logits to produce probability. Loss function is BCEWithLogitsLoss with pos_weight=0.23 to handle class imbalance. Optimizer is Adam with learning rate 0.001 and ReduceLROnPlateau scheduler.

MagLSTM Architecture - Regression model with similar architecture but wider input layer (26 features × 5 timesteps) and different output layer. Output is magnitude prediction (continuous value). Loss function is MSELoss. Other hyperparameters are identical to TimeLSTM.

Training Strategy - Both models use early stopping with patience=15 epochs to prevent overfitting. Learning rate is automatically reduced when validation loss stops improving. Model checkpoints are saved at the best epoch. Sequence length=5 allows models to look back at the 5 most recent events to predict the next event.

7.3 Evaluation & Visualization System

Metrics Tracking - During training, metrics are tracked for both training and validation sets. TimeLSTM tracks loss, accuracy, AUC, and F1 score. MagLSTM tracks loss, MAE, and RMSE. Metrics are logged each epoch and used to determine the best model.

Test Set Evaluation - After training completion, the best model is loaded and evaluated on the test set. Metrics are calculated and compared with training/validation metrics to assess generalization capability.

Visualization - Training curves are plotted to monitor the learning process. Test set predictions are visualized with scatter plots (predicted vs actual) and error distribution plots. These visualizations help understand model behavior and limitations.

7.4 Deployment Interface

Web Application - Flask-based web interface enables interactive prediction. Users can select a seismic zone from a list of highly active regions (Japan, Philippines, Chile, etc.), enter current event information (magnitude, depth, location), and receive predictions about timing and magnitude of the next earthquake.

Model Serving - Trained models are loaded together with scalers and region encoders to ensure consistent preprocessing. Predictions are returned as JSON with probability scores for TimeLSTM and magnitude estimates with uncertainty bounds for MagLSTM.

8. STRENGTHS AND WEAKNESSES

8.1 TimeLSTM

Strengths: Excellent class imbalance handling with pos_weight (0.23), probabilistic output enables flexible threshold tuning, LSTM effectively learns temporal patterns with sequence_length=5, stable training with minimal overfitting (train-test gap ~2.5%), AUC 0.9016 is excellent performance, Accuracy 81.47% is very high for binary classification task.

Weaknesses: Binary limitation (yes/no within 7 days), fixed 7-day window may not be optimal for all regions, false negatives are dangerous in real-world deployment, class imbalance 4:1 remains a challenge despite handling.

8.2 MagLSTM

Strengths: Continuous output for magnitude prediction with MAE 0.461, spatial features (latitude, longitude, regional_b_value) capture regional patterns, b-value is a physics-based feature with practical significance, good for magnitude range 1.0-2.0, small train-test gap (0.40 → 0.46) indicates reasonable generalization.

Weaknesses: MAE 0.461 magnitude is still relatively large (33% of mean target), no uncertainty quantification for predictions, outlier sensitivity for extreme events (RMSE > MAE), slight overfitting during training (test loss 0.341 > val loss 0.258), does not ensure physical constraints (magnitude cannot be negative).

9. CONCLUSION AND FUTURE DIRECTIONS

This project has achieved significant milestones: complete ML pipeline from raw data to deployed models, 31 advanced features with physical meanings, dual LSTM architecture optimized for two different tasks, comprehensive evaluation with excellent test metrics (TimeLSTM: 81.47% accuracy, 0.9016 AUC; MagLSTM: 0.461 MAE), production-ready Flask web application, and comprehensive documentation.

Applications: TimeLSTM can be used for early warning systems with probability thresholds customizable according to risk tolerance. MagLSTM is suitable for impact estimation with magnitude ranges, aiding better planning and response. Combining both models enables prediction of both timing and magnitude of the next earthquake.

Future Directions: Real-time inference pipeline with streaming data, regional models for different seismic zones, multi-horizon prediction (not just 7 days but 3, 14, 30 days), uncertainty quantification with Bayesian methods or ensemble models, attention mechanism to replace or supplement LSTM, physics-informed neural networks to ensure predictions satisfy physical constraints.
